

# ფოტოების გამოსწორების გეგმა

## პრობლემა
`getOptimizedImageUrl()` ფუნქცია Supabase URL-ებს `/render/image/public/` endpoint-ზე გადამისამართებს, რომელიც **მხოლოდ Supabase Pro გეგმაზე** მუშაობს. უფასო გეგმაზე ყველა სურათი იტეხება (404/400 error).

## გადაწყვეტა
`getOptimizedImageUrl` ფუნქციაში `/render/image/` ტრანსფორმაცია უნდა მოვხსნათ და ორიგინალი `/object/public/` URL დავტოვოთ. კლიენტის მხარეს კომპრესია (ატვირთვისას) უკვე მუშაობს — ეს საკმარისია ახალი ფოტოებისთვის. ძველი ფოტოებისთვის კი ორიგინალი URL-ი უნდა ჩაიტვირთოს.

## ცვლილებები

### ფაილი: `src/utils/imageCompression.ts`
- `getOptimizedImageUrl` ფუნქცია: `/render/image/` ტრანსფორმაციის მოხსნა
- ფუნქცია უბრალოდ ორიგინალ URL-ს დააბრუნებს (რადგან Pro გეგმა არ არის)
- კლიენტის მხარეს `compressImage` ფუნქცია რჩება — ახალი ფოტოები WebP-ში იტვირთება

### რატომ ეს მიდგომა
- **ატვირთვისას კომპრესია** უკვე დანერგილია და მუშაობს (4MB → 200-400KB)
- სერვერის მხრიდან ტრანსფორმაცია (render/image) Pro გეგმას მოითხოვს
- ორიგინალი URL-ის გამოყენება ყველა ფოტოს აღადგენს

### გავლენა
- ServiceCard, ServiceGallery — ფოტოები დაუბრუნდება
- ახალი ატვირთვები — კვლავ WebP + კომპრესირებული
- ძველი ფოტოები — ორიგინალი ზომით ჩაიტვირთება (სანამ ხელახლა არ აიტვირთება)
